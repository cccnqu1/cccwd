實驗結果

我用 JavaScript 代替中文進行 facebook 溝通，發現其實要以 JavaScript 語法進行交談是完全沒問題的。

這代表人與人可以透過程式語言溝通！

而程式語言 (Javascript) 原本就是人與電腦溝通的工具。

這樣的話，是否代表程式語言可以順利地串聯人與電腦呢？

並非如此，因為我在「人與人之間用 javascript 溝通時，並沒有寫出完整的語意，因此這些程式的執行結果很多都沒有意義」。

因此我們只是採用 javascript 的語法，編碼人類的語言，然後傳送出去，接著對方接到這種 javascript 語言之後，在解碼回人類的語言，理解其意義而已。

在本實驗中，我們發現「javascript 語法絕對可以承載人類的語言訊息，但是如何讓電腦理解這些語言訊息的語意，仍然是一個難以克服的問題」。

如果要進一步的實驗「人與人，人與機器」之間的一致性溝通，寫出來的語法必須具有讓電腦也能執行正確動作，或者回應正確訊息的語意才行。

這又回到了自然語言理解問題，不過有個好處，剖析 (parse) 問題將變得比較小，但是語意問題仍然存在。
